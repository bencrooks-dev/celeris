# 2-D Tensor Memory Model Plan (v0.5.0)

> **Plan ID:** `plan:tensor-2d`
> **For agentic workers:** `/athena-build` wave-by-wave. TDD per task. Branch `feat/tensor-2d`. Venv `/Users/bencrooks/celeris/.venv/bin/python`.

**Goal:** Support 2-D arrays with `a[i, j]` element indexing and **general strides** (NumPy views/transposes work natively), via `F64Array2D`-style annotations.

**Approach:** A 2-D array param has IR type `{"ptr": elem, "ndim": 2}` (1-D stays `{"ptr": elem}`, `ndim` defaults to 1). A multi-dim index is a new IR node `{"k":"index","array","indices":[e0,e1],"type":elem}` (1-D `index`/`lval_index` with single `"index"` are unchanged → full back-compat). Native backends pass each 2-D array as `data_ptr + one int64 stride per dim` (strides in elements); `a[i,j]` lowers to `data[i*s0 + j*s1]`. The interpreter indexes the real NumPy array natively (the oracle). Slicing and broadcasting are out of scope (v0.6).

**Tech Stack:** Python 3.10+, NumPy strides, C++17, llvmlite, ctypes, pytest, ruff.

**Files:**
- Modify: `src/celeris/types.py` — `F64Array2D`/`F32Array2D`/`I64Array2D`/`I32Array2D` markers → `{"ptr":elem,"ndim":2}`; `ndim_of(t)` helper
- Modify: `src/celeris/ir.py` — `index_nd(array, indices, type)`, `lval_index_nd(array, indices, type)`
- Modify: `src/celeris/parser.py` — allow non-slice tuple subscripts; emit `index_nd` for N-D arrays with arity = ndim
- Modify: `src/celeris/verifier.py` — accept `ndim` on ptr types; verify `index_nd` (arity == ndim, int indices, elem type)
- Modify: `src/celeris/backends/interpreter.py` — `index_nd` via native NumPy multi-index
- Modify: `src/celeris/backends/sourcegen.py` — strided signature + offset + marshalling
